Introducción a Quarto.

Temario

1. Introducción a Quarto:

  • ¿Qué es Quarto? (5 minutos)
    • Definición de Quarto como un lenguaje de markup para la creación de documentos.
    • Diferencias entre Quarto y RMarkdown.
    • Ventajas de usar Quarto: flexibilidad, portabilidad, multiformato.
  • ¿Qué se puede crear con Quarto? (5 minutos)
    • Mostrar ejemplos de documentos Quarto: informes, artículos, sitios web, libros, presentaciones.
    • Demostrar la versatilidad de Quarto para diferentes áreas: ciencia, academia, educación, negocios.

2. Primeros pasos con Quarto:

  • Instalación y configuración (10 minutos)
    • Explicar cómo instalar Quarto y RStudio.
    • Crear un nuevo proyecto Quarto en RStudio.
    • Familiarizar al público con la interfaz de RStudio para Quarto.
  • Sintaxis básica de Markdown (15 minutos)
    • Introducir los elementos básicos de Markdown: encabezados, listas, código, imágenes, etc.
    • Mostrar ejemplos de código Markdown y su resultado visual.
    • Practicar la escritura de Markdown en un editor de texto.

3. Funciones avanzadas de Quarto:

  • Formatos de salida (10 minutos)
    • Mostrar cómo generar diferentes formatos de salida a partir de un documento Quarto: HTML, PDF, EPUB, etc.
    • Demostrar la configuración de opciones de salida específicas para cada formato.
  • Plantillas y temas (10 minutos)
    • Explicar el uso de plantillas para personalizar la apariencia de los documentos.
    • Mostrar cómo crear y usar temas para aplicar estilos predefinidos.
  • Chunks de código R (15 minutos)
    • Introducir los chunks de código R para ejecutar código dentro de un documento Quarto.
    • Mostrar cómo insertar gráficos, tablas y otros resultados de R en un documento.
    • Demostrar la interacción entre código R y Markdown.

4. Recursos y ejemplos:

  • Documentación oficial de Quarto (5 minutos)
    • Presentar la página web oficial de Quarto como fuente de información y tutoriales.
    • Resaltar la disponibilidad de guías y ejemplos para diferentes casos de uso.
  • Comunidad Quarto (5 minutos)
    • Mencionar la existencia de una comunidad activa de usuarios de Quarto.
    • Mostrar ejemplos de proyectos Quarto creados por la comunidad.
    • Brindar información sobre cómo unirse a la comunidad y obtener ayuda.

5. Práctica y preguntas:

  • Tiempo para practicar (20 minutos)
    • Proponer un ejercicio práctico para que los participantes apliquen lo aprendido.
      • Un reporte de semestre
      • Brindar asistencia y soporte durante la práctica.
  • Ronda de preguntas y respuestas (15 minutos)
    • Responder a las preguntas de los participantes sobre Quarto y su uso.
    • Brindar información adicional y recomendaciones.

Cierre:

  • Resumen de los puntos clave (5 minutos)
  • Agradecimiento a los participantes (2 minutos)
  • Invitación a seguir aprendiendo y explorando Quarto (2 minutos)
